# CLAUDE.md 템플릿

## 핵심 원칙

- **간결성**: 밀도 높고 사람이 읽기 편한 콘텐츠. 가능하면 한 개념당 한 줄로 구성
- **조치 가능성**: 복사-붙여넣기가 즉시 가능한 명령어 형태
- **프로젝트 특화**: 일반적인 조언이 아닌 이 프로젝트만의 고유한 패턴 문서화
- **최신성**: 모든 정보는 실제 코드베이스 상태를 반영해야 함

---

## 권장 섹션

프로젝트와 관련된 섹션만 사용하십시오. 모든 섹션이 다 필요한 것은 아닙니다.

### 명령어 (Commands)

프로젝트 작업에 필수적인 명령어를 문서화합니다.

```markdown
## 명령어

| 명령어 | 설명 |
|---------|-------------|
| `<설치 명령어>` | 종속성 설치 |
| `<개발 서버 실행 명령어>` | 개발 서버 시작 |
| `<빌드 명령어>` | 프로덕션 빌드 |
| `<테스트 실행 명령어>` | 테스트 실행 |
| `<린트/포맷 명령어>` | 코드 린트/포맷 |
```

### 아키텍처 (Architecture)

Claude가 구성 요소들의 위치를 이해할 수 있도록 프로젝트 구조를 설명합니다.

```markdown
## 아키텍처

```
<root>/
  <dir>/    # <용도/목적>
  <dir>/    # <용도/목적>
  <dir>/    # <용도/목적>
```
```

### 주요 파일 (Key Files)

Claude가 알고 있어야 하는 중요 파일 목록을 작성합니다.

```markdown
## 주요 파일

- `<경로>` - <용도/목적>
- `<경로>` - <용도/목적>
```

### 코드 스타일 (Code Style)

프로젝트 특화 코딩 컨벤션을 문서화합니다.

```markdown
## 코드 스타일

- <컨벤션/규칙>
- <컨벤션/규칙>
- <특정 대안보다 선호하는 방식>
```

### 환경 설정 (Environment)

필요한 환경 변수와 설정 단계를 문서화합니다.

```markdown
## 환경 설정

필수:
- `<변수명>` - <용도/목적>
- `<변수명>` - <용도/목적>

설정:
- <설정 단계>
```

### 테스트 (Testing)

테스트 접근 방식과 명령어를 문서화합니다.

```markdown
## 테스트

- `<테스트 명령어>` - <테스트 대상>
- <테스트 규칙 또는 패턴>
```

### 주의 사항 (Gotchas)

직관적이지 않은 패턴, 특이 사항 및 경고를 문서화합니다.

```markdown
## 주의 사항

- <문제를 유발할 수 있는 직관적이지 않은 요소>
- <순서 의존성 또는 선수 조건>
- <피해야 할 흔한 실수>
```

### 워크플로우 (Workflow)

개발 워크플로우 패턴을 문서화합니다.

```markdown
## 워크플로우

- <X를 해야 하는 시점>
- <Y에 대해 선호하는 접근 방식>
```
